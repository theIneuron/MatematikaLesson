# SETUP — как развернуть этот pipeline в Claude Code

Это пошаговая инструкция. Команды можно копировать как есть. Если шаг не сработал —
в конце каждого блока есть ссылка на официальную доку.

---

## Что в этой папке

```
coddycamp-math/
├── CLAUDE.md                 ← инструкции (Claude Code читает их сам при старте)
├── SETUP.md                  ← этот файл
└── .claude/
    └── skills/               ← 6 скиллов pipeline, каждый в своей папке
        ├── program-architect/SKILL.md
        ├── skeleton-generator/SKILL.md
        ├── content-generator/SKILL.md
        ├── jsx-builder/SKILL.md
        ├── qa-validator/SKILL.md
        └── knowledge-updater/SKILL.md
```

Это скиллы, **активные сейчас в Project** — то, что реально работает. Если ты где-то
держишь более свежую версию какого-то скилла (правил через настройки Skills уже после
последней синхронизации) — скажи, какой именно, и я подтяну её.

## Шаг 1. Установить Claude Code

Нужен Node.js (версия 18+). Проверь, есть ли он:

```bash
node --version
```

Если выдало номер версии (например `v20.x`) — Node есть, иди дальше. Если "command
not found" — поставь Node с nodejs.org (LTS-версия), затем установи Claude Code:

```bash
npm install -g @anthropic-ai/claude-code
```

Проверка:

```bash
claude --version
```

Доку по установке (есть и установщик без npm): https://docs.claude.com/en/docs/claude-code/overview

---

## Шаг 2. Открыть папку и запустить

Положи папку `coddycamp-math` куда удобно (например в Документы). Зайди в неё и
запусти Claude Code:

```bash
cd путь/к/coddycamp-math
claude
```

При первом запуске попросит войти в аккаунт Anthropic — войди тем же, что и на
claude.ai. Claude Code прочитает `CLAUDE.md` и подхватит скиллы из `.claude/skills/`
автоматически.

Проверить, что скиллы видны: в запущенном Claude Code набери `/help` (там будет
раздел про skills) или просто напиши **"покажи доступные скиллы"**. Должны быть все
шесть.

---

## Шаг 3. Подключить Notion (без этого скиллы не работают)

Knowledge base (методика, audio_rules, эталон, программа — всё) лежит в Notion.
Скиллы читают её по ID страниц. В Claude Code доступ даёт Notion MCP. **Пока он не
подключён — скиллы запускаться будут, но валиться на чтении базы.**

Самый простой путь — добавить hosted-сервер Notion и авторизоваться через браузер:

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp
```

Затем **перезапусти** Claude Code (выйди командой `exit`, запусти `claude` снова —
новые MCP подхватываются только при старте). Внутри набери:

```
/mcp
```

В списке появится `notion` — пройди OAuth в открывшемся окне и дай доступ к рабочему
пространству, где лежат **Platform Standards DOCS** и **Math Lessons Pipeline DOCS**.

Альтернатива — официальный плагин Notion для Claude Code (ставит MCP + готовые
команды в один клик): https://github.com/makenotion/claude-code-notion-plugin
Доку Notion по подключению: https://developers.notion.com/guides/mcp/get-started-with-mcp

Проверка: попроси **"прочитай страницу teaching_methodology по ID
37356053874681ed9ec1cc5866f95ae6"**. Если Claude вернул содержимое — связь есть, всё
готово к работе.

## Шаг 4. Как смотреть готовый урок (.jsx) — главное отличие от claude.ai

В Claude Code **нет** панели артефактов, как на claude.ai. `jsx-builder` сохранит
урок как файл `.jsx`, но прокликать его прямо в терминале нельзя. Три варианта, от
простого к сложному:

**Вариант A (рекомендую для старта, ноль настройки).**
Генерируешь и проверяешь урок в Claude Code, а сам `.jsx` открываешь на claude.ai:
создаёшь там артефакт и вставляешь код — урок отрендерится и кликается как обычно.
Гибрид: тяжёлую работу с файлами и базой делает Claude Code, превью — веб.

**Вариант B (превью локально, одна команда).**
В отдельном терминале:

```bash
npx run-claude-artifact путь/к/уроку.jsx
```

Откроется вкладка браузера с работающим уроком. Останавливается через Ctrl+C.

**Вариант C (полноценный проект).**
Поднять Vite-проект с React и подключать туда уроки. Это уже разработческий setup —
есть смысл, только если уроков много и нужен один стенд для всех. Claude Code сам
поможет его собрать, если решишь идти этим путём.

## Стоит ли вообще переезжать в Claude Code — честно

Плюсы для этого pipeline: Claude Code сильнее в работе с файлами, а `qa-validator`
гоняет bash-проверки регулярками — в терминале это работает нативно и чище.

Минус один, но ощутимый: это терминальный инструмент. Установка, MCP, превью через
сервер — для не-разработчика это лишние слои поверх того, что на claude.ai делается
кликом.

**Если терминал кажется тяжёлым — посмотри Claude Cowork** (десктоп-приложение для
не-разработчиков). Он использует те же скиллы и тот же Notion, но без терминала, и в
нём есть превью артефактов — то есть `.jsx` можно прокликать прямо в приложении.
По задаче "методист гоняет pipeline уроков" Cowork часто подходит лучше, чем Claude
Code. Скиллы из этой папки переносятся в него так же. Уточни перед переездом, как в
твоей версии Cowork загружаются кастомные скиллы (через настройки).

Реши сам — пакет одинаково ложится и в Claude Code, и в Cowork.
